# OGPと招待ページの充実化

作成日: 2026-06-12

## 概要

相手募集機能は見送り、代わりにOGPと招待リンクページの充実化に注力する方針。
ユーザーが自発的にSNSでシェアする際の体験を最大化し、個々の交換日記の拡散を支援する。

---

## 方針の背景

### 相手募集機能を見送る理由

1. **現状のユーザー数では逆効果**
   - 募集一覧ページが閑散として見える
   - 「誰も使っていない感」が出てしまう
   - プラットフォーム全体の印象を損なうリスク

2. **ユーザー主導の方が自然**
   - 既存のSNSで募集する方が自由度が高い
   - 各自のコミュニティで呼びかける方が信頼性がある
   - プラットフォーム側が仲介する必要性が薄い

### OGP・招待ページ充実化を優先する理由

1. **個々の拡散を支援する**
   - ユーザーが自発的にSNSでシェアする際の体験を最大化
   - 魅力的なOGPカードで興味を引ける
   - 招待リンクを受け取った人の初見体験が重要

2. **プラットフォーム全体の見栄えが良くなる**
   - 交換日記ごとの個性が伝わる
   - TwitterやDiscordでのシェア時の印象が向上
   - ブックマーク時の視認性も改善

3. **集中すべき投資先として効率的**
   - 「募集機能」は複雑なマッチングシステムが必要
   - 一方、OGP・招待ページは明確な成果が見込める
   - 実装コストに対するリターンが大きい

---

## 実装内容

### 1. OGPの作り込み

現在のOGP実装（ユーザーページで実装済み）をさらに拡張し、交換日記と投稿のページでも充実させる。

#### 交換日記（Thread）詳細ページ

**meta_title**

```
[交換日記のタイトル] - coconikki
```

**meta_description**

```
[交換日記の説明文（最初の100文字程度）]
```

**meta_image**

- 交換日記に設定した画像
- 画像が設定されていない場合はデフォルトのOGP画像

**meta_type**

```
article
```

**twitter_card**

```
summary
```

#### 投稿（Post）詳細ページ

**meta_title**

```
[投稿者の表示名] さんの投稿 - [交換日記のタイトル] - coconikki
```

**meta_description**

```
[投稿本文の冒頭100文字程度のプレーンテキスト]
```

**meta_image**

- 画像付き投稿なら最初の画像をOGP画像に
- 画像がない場合は交換日記のOGP画像
- 交換日記のOGP画像もない場合はデフォルトOGP画像

**meta_type**

```
article
```

**twitter_card**

```
summary_large_image（画像付き投稿の場合）
summary（画像がない場合）
```

#### ユーザーページ（実装済み）

**meta_title**

```
[表示名] (@username) - coconikki
```

**meta_description**

```
[表示名] さんのcoconikkiページです。参加中の交換日記や投稿をご覧いただけます。
```

**meta_image**

- ユーザーのアバター画像（600x600にリサイズ）
- アバターがない場合はデフォルト画像

**twitter_card**

```
summary
```

### 2. 招待リンクページの充実化

現在の `/invite/:token` ページを魅力的なランディングページに改善。

#### デザイン方針

**魅力的なランディングページ化**

- 交換日記のタイトル・説明を大きく表示
- 参加者一覧（アバター付き）
- 「参加する」ボタンを目立たせる

**参加前のプレビュー機能**

- 公開スレッドなら最新投稿をいくつか表示（タイトル・冒頭部分）
- 「どんな雰囲気か」が伝わるようにする

**招待者の意図を尊重**

- 招待者が添えたメッセージがあれば表示（将来的な拡張）
- 「〇〇さんがあなたを招待しました」と表示

#### 技術的な実装

- 既存の `InvitationsController#show` を拡張
- 招待トークンから交換日記情報を取得
- 参加者一覧、最新投稿（公開の場合）を取得
- OGPメタタグも設定（招待ページ自体もシェアされる可能性がある）

---

### 3. シェア機能の追加

#### 「招待リンクをコピー」ボタン

**配置場所**

- スレッド設定ページ（`/threads/:slug/edit`）
- スレッド詳細ページのサイドバー（オプション）

**機能**

- ワンクリックでクリップボードにコピー
- トースト通知「招待リンクをコピーしました」
- Stimulus コントローラーで実装

#### 「SNSでシェア」ボタン（オプション）

**配置場所**

- スレッド詳細ページ
- 投稿詳細ページ

**機能**

- Twitter/Xシェアボタン
- あらかじめテキストを入れておく: 「交換日記『[タイトル]』をcoconikkiで書いています」
- URLはスレッド詳細ページまたは投稿詳細ページ

## 実装の優先順位

### Phase 1: OGP強化（1週間程度）

**目標**: 交換日記と投稿のOGPを実装し、SNSでのシェア体験を向上

1. **交換日記詳細ページのOGP実装**
   - `app/views/threads/show.html.slim` にメタタグ設定を追加
   - `content_for :meta_title`, `:meta_description`, `:meta_image`, `:meta_type`, `:twitter_card`
   - `ThreadsController#show` で必要な情報を用意

2. **投稿詳細ページのOGP実装**
   - `app/views/threads/posts/show.html.slim` にメタタグ設定を追加
   - 投稿本文から `extract_plain_text` でdescriptionを生成
   - 画像添付がある場合は最初の画像をOGP画像に

3. **動作確認**
   - Twitter Card Validator で確認
   - Discord, Slack などでの表示確認

---

### Phase 2: 招待ページ改善（1-2週間）

**目標**: 招待リンクを受け取った人の初見体験を最大化

1. **`/invite/:token` ページのデザイン刷新**
   - 新しいビューファイルを作成または既存を大幅改修
   - 交換日記の情報を魅力的に表示
   - 参加者一覧（アバター付き）
   - 「参加する」ボタンを目立たせる

2. **プレビュー機能の追加**
   - 公開スレッド: 最新投稿3-5件のタイトルと冒頭を表示
   - 「どんな交換日記か」が伝わるようにする

3. **招待ページのOGP設定**
   - 招待ページ自体もシェアされる可能性があるため、OGPを設定
   - タイトル: 「[交換日記名]への招待 - coconikki」
   - 説明: 交換日記の説明文
   - 招待者が意図してシェアするならいいが、検索結果などには出したくないので noindexを設定する必要がある

4. **参加フローのUX改善**
   - 未ログイン状態での招待リンクアクセス時のフロー改善
   - ログイン後に招待ページに戻る導線
   - 参加完了後のフィードバック

---

### Phase 3: シェア機能（数日）

**目標**: ユーザーが簡単に招待リンクをシェアできるようにする

1. **招待リンクのコピーボタン**
   - スレッド設定ページに追加
   - Stimulus コントローラーで実装
   - クリップボードAPIを使用
   - トースト通知でフィードバック

2. **SNSシェアボタン（オプション）**
   - Twitter/Xシェアボタン
   - シェアテキストのテンプレート作成
   - スレッド詳細ページに配置

---

## この方針の利点

1. **スモールスタート**
   - 大掛かりな機能開発を避けられる
   - 段階的に実装できる

2. **ユーザー主導**
   - プラットフォームが仲介せず、各自の裁量で拡散
   - ユーザーの自主性を尊重

3. **成長への布石**
   - 将来的にユーザーが増えたら募集機能を追加する選択肢は残る
   - 今は基盤を固める

4. **即効性**
   - OGPとランディングページの改善は今すぐ効果が見込める
   - シェア時の印象が劇的に向上

5. **ブランディング**
   - 魅力的なシェア体験はcoconikkiの印象を向上させる
   - 口コミでの拡散を促進

## 技術的な参考情報

### OGP実装のパターン（すでに実装済み）

**ユーザーページの例** (`app/views/users/show.html.slim`)

```slim
/ メタタグ設定
- content_for :meta_title, "#{@user.display_name} (@#{@user.username}) - coconikki"
- content_for :meta_description, "#{@user.display_name} さんのcoconikkiページです。参加中の交換日記や投稿をご覧いただけます。"
- content_for :twitter_card, "summary"
- if @user.avatar.attached? && @user.avatar.representable?
  - content_for :meta_image, polymorphic_url(@user.avatar.representation(resize_to_limit: [600, 600]))
```

**MetaTagsHelper** (`app/helpers/meta_tags_helper.rb`)

- `meta_title`, `meta_description`, `meta_image`, `meta_url`, `meta_type`, `meta_twitter_card` ヘルパーメソッド
- `extract_plain_text(markdown_text, max_length: 100)` - マークダウンからプレーンテキストを抽出

**レイアウトファイル** (`app/views/layouts/application.html.slim`)

- OGPメタタグとTwitter Cardメタタグがすでに設定済み
- 各ページで `content_for` を使って値を設定すれば自動的に反映される

### 投稿の画像取得

投稿に添付された画像を取得する場合：

```ruby
@post.images.first # 最初の画像
@post.images.first.representation(resize_to_limit: [1200, 630]) # OGP用にリサイズ
polymorphic_url(@post.images.first.representation(resize_to_limit: [1200, 630])) # URL生成
```
