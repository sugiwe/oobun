# 非アクティブユーザー自動削除機能（フェーズ2）

作成日: 2026-04-06

## 概要

完全オープンログイン化に伴い、非アクティブユーザーを自動削除する仕組みを導入する。
ユーザーの活動状況に応じて削除期間を調整し、適切なタイミングで削除を実施する。

---

## 削除ポリシー

### パターンA: 何もしてないユーザー（厳しめ）

**対象**:
- 登録後、投稿・フォロー・交換日記参加が一切ないユーザー

**削除期間**:
- **90日間**

**理由**:
- お試しだけして放置されたアカウント
- サーバーリソースの無駄遣いを防ぐ

**通知タイミング**:
- 75日目: 削除予告メール送信
- 90日目: アカウント削除実行

---

### パターンB: 活動歴があるユーザー（緩め）

**対象**:
- 投稿・フォロー・交換日記参加のいずれかがあるユーザー

**削除期間**:
- **1年間ログインなし**

**理由**:
- 一度は使ってくれた人だから長めに猶予を与える
- 再訪の可能性を考慮

**通知タイミング**:
- 345日目: 削除予告メール送信
- 365日目: アカウント削除実行

---

## 実装方針

### 1. データベース変更

**User テーブルに追加するカラム**:
```ruby
add_column :users, :last_signed_in_at, :datetime
```

**用途**:
- ログイン日時を記録
- 非アクティブ判定の基準とする

---

### 2. モデルの実装

**User モデルに追加するスコープ**:

```ruby
# 何もしてない非アクティブユーザー（90日間）
scope :inactive_new_users, -> {
  where('created_at < ?', 90.days.ago)
    .left_joins(:posts, :subscriptions, :memberships)
    .where(posts: { id: nil }, subscriptions: { id: nil }, memberships: { id: nil })
}

# 活動歴があるが長期間ログインなし（1年間）
scope :inactive_active_users, -> {
  where('last_signed_in_at < ?', 1.year.ago)
    .where.not(id: inactive_new_users.select(:id))
}

# 削除予告対象（パターンA: 75日目）
scope :warning_new_users, -> {
  where('created_at BETWEEN ? AND ?', 76.days.ago, 75.days.ago)
    .left_joins(:posts, :subscriptions, :memberships)
    .where(posts: { id: nil }, subscriptions: { id: nil }, memberships: { id: nil })
}

# 削除予告対象（パターンB: 345日目）
scope :warning_active_users, -> {
  where('last_signed_in_at BETWEEN ? AND ?', 346.days.ago, 345.days.ago)
    .where.not(id: inactive_new_users.select(:id))
}
```

---

### 3. Rake タスクの作成

**lib/tasks/user_cleanup.rake**:

```ruby
namespace :users do
  desc "非アクティブユーザーに削除予告メールを送信"
  task send_deletion_warnings: :environment do
    # パターンA: 何もしてないユーザー（75日目）
    User.warning_new_users.find_each do |user|
      UserMailer.deletion_warning_new_user(user).deliver_later
      Rails.logger.info "Deletion warning sent to new user: #{user.email}"
    end

    # パターンB: 活動歴があるユーザー（345日目）
    User.warning_active_users.find_each do |user|
      UserMailer.deletion_warning_active_user(user).deliver_later
      Rails.logger.info "Deletion warning sent to active user: #{user.email}"
    end
  end

  desc "非アクティブユーザーを削除"
  task cleanup_inactive: :environment do
    # パターンA: 何もしてないユーザー（90日）
    new_users = User.inactive_new_users
    new_count = new_users.count
    new_users.find_each do |user|
      user.update!(deleted_at: Time.current) # 論理削除
      Rails.logger.info "Deleted inactive new user: #{user.email}"
    end

    # パターンB: 活動歴があるユーザー（1年）
    active_users = User.inactive_active_users
    active_count = active_users.count
    active_users.find_each do |user|
      user.update!(deleted_at: Time.current) # 論理削除
      Rails.logger.info "Deleted inactive active user: #{user.email}"
    end

    puts "#{new_count}人の新規非アクティブユーザーと#{active_count}人の活動歴ありユーザーを削除しました"
  end
end
```

### 5. メーラーの実装

**UserMailer の追加メソッド**:

```ruby
class UserMailer < ApplicationMailer
  def deletion_warning_new_user(user)
    @user = user
    @days_remaining = 15
    mail(
      to: user.email,
      subject: "【coconikki】アカウント削除予告のお知らせ"
    )
  end

  def deletion_warning_active_user(user)
    @user = user
    @days_remaining = 20
    mail(
      to: user.email,
      subject: "【coconikki】長期未ログインによるアカウント削除予告"
    )
  end
end
```

---

### 6. cron 設定

**毎日実行**:

```bash
# 毎日午前3時に削除予告メール送信
0 3 * * * cd /path/to/app && bin/rails users:send_deletion_warnings

# 毎日午前4時に非アクティブユーザー削除
0 4 * * * cd /path/to/app && bin/rails users:cleanup_inactive
```

## アカウントの削除について

以下の場合、予告なくアカウントを削除する場合があります：

### 1. 登録後90日間、活動がない場合
- 投稿、交換日記への参加、フォローのいずれも行っていない場合

- 過去に投稿や交換日記への参加実績があるユーザーが対象

- 利用規約に違反する行為が確認された場合

削除されたアカウントのデータは、プライバシー保護のため速やかに削除されます。

```markdown
## 個人情報の保存期間

### アクティブユーザー
- アカウントが有効な間、個人情報を保存します

### 非アクティブユーザー
- 登録後90日間活動がない場合、通知の上でアカウントを削除します
- 1年間ログインがない場合、通知の上でアカウントを削除します

### 削除後
- アカウント削除後、個人情報は速やかに削除されます
- ただし、法令で保存が義務付けられている情報は除きます
```

## 注意事項

- 論理削除（`deleted_at`）を使用するため、物理削除は行わない
- 削除予告メールは必ず送信すること（ユーザー体験の向上）
- cron の実行時間帯はサーバー負荷を考慮すること
- 本番環境での初回実行前に、ステージング環境でテストすること
