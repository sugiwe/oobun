# バックアップシステム ドキュメント

coconikkiのバックアップシステムに関するドキュメント一覧です。

---

## 📚 ドキュメント構成

### 1. [運用マニュアル (operations.md)](operations.md)

**対象**: 日常的にバックアップを運用する方

**内容**:
- バックアップシステムの概要
- 自動バックアップの確認方法
- 手動バックアップの実行方法
- 簡易的な復元手順
- トラブルシューティング
- ファイル構成

**いつ読む**:
- 日常的な運用で困ったとき
- バックアップの状態を確認したいとき
- 緊急時のクイックリファレンスとして

---

### 2. [復元ガイド (restore-guide.md)](restore-guide.md)

**対象**: バックアップからデータを復元する必要がある方

**内容**:
- バックアップファイルの詳細説明
- 復元の仕組みとフロー
- 復元手順（最新/特定日時）
- 復元後の確認方法
- トラブルシューティング
- 緊急時の手順
- ベストプラクティス

**いつ読む**:
- データが失われて復元が必要なとき
- 復元のテストをしたいとき
- バックアップファイルの中身を確認したいとき

---

### 3. [セットアップ手順 (setup.md)](setup.md)

**対象**: 新しいサーバーでバックアップシステムを構築する方

**内容**:
- VPSへのrcloneインストール
- Google Drive OAuth認証設定
- バックアップスクリプトの配置
- Discord Webhook設定
- cron設定（自動実行）
- 動作確認

**いつ読む**:
- 新しいVPSでバックアップを設定するとき
- 別の環境に同じシステムを構築したいとき
- セットアップ手順を確認したいとき

## 🚀 はじめに読むべきドキュメント

### ケース1: バックアップが正常に動いているか確認したい
→ [運用マニュアル](operations.md) の「日常運用」セクション

### ケース2: データが消えた！復元したい！
→ [復元ガイド](restore-guide.md) の「復元手順」セクション

### ケース3: 新しいサーバーでバックアップを設定したい
→ [セットアップ手順](setup.md) を最初から順番に

### ケース4: Discord通知が来ない
→ [運用マニュアル](operations.md) の「トラブルシューティング」セクション

### ケース5: バックアップファイルの中身を見たい
→ [復元ガイド](restore-guide.md) の「バックアップファイルの中身」セクション

## 📞 サポート

ドキュメントを読んでも解決しない場合は、以下の情報を用意して相談してください：

1. **発生している問題**
   - 何をしようとしたか
   - 何が起きたか（エラーメッセージ等）

2. **ログ**
   ```bash
   # バックアップログ
   tail -100 ~/backup.log

   # アプリケーションログ
   cd /home/deploy/oobun
   bin/kamal app logs
   ```

3. **環境情報**
   ```bash
   # rcloneバージョン
   rclone version

   # cron設定
   crontab -l

   # 環境変数
   echo $DISCORD_WEBHOOK_URL
   ```
